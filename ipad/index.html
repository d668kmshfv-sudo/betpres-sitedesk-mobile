Bezpečné skúšanie troch používateľov

Každé cloudové uloženie kontroluje verziu. Ak dvaja používatelia upravia rovnakú verziu súčasne, druhé uloženie sa zastaví a neprepíše novšie dáta.

Čo treba urobiť v Supabase

  1. Vytvoriť nový Supabase projekt na svojom účte.
  2. V existujúcom projekte otvoriť SQL Editor a spustiť súbor SUPABASE_TEAM_PILOT.sql.
  3. V Project Settings → API skopírovať Project URL a publishable/anon key.
  4. Každý z troch používateľov si vytvorí vlastný účet; vlastník potom pridá kolegov e-mailom.
  5. Prvýkrát stlačiť Odoslať lokálne dáta do cloudu.
